# 基础物理与平台碰撞说明

## 1. 目标

第 13 步实现最小可玩的 2D 平台物理：玩家可以跳起、受重力下落，并在从上方落到平台顶面时停住。该逻辑服务核心玩法闭环，不依赖 Cocos 物理引擎。

## 2. 坐标约定

- `position.x` 表示玩家脚底中心的水平坐标。
- `position.y` 表示玩家脚底的垂直坐标。
- 屏幕坐标向下增大。
- 跳跃速度向上，因此 `velocity.y` 为负数。
- 重力向下，因此每帧会让 `velocity.y` 增大。

## 3. 配置字段

关卡配置中的 `physics` 字段当前包含：

- `gravity`：重力加速度。
- `maxFallSpeed`：最大下落速度。
- `playerBounds.width`：玩家碰撞体宽度。
- `playerBounds.height`：玩家碰撞体高度，当前主要用于记录和后续扩展。

平台数据来自 `platforms`，每个平台的 `y` 表示平台顶面。

## 4. 当前碰撞范围

当前只实现从上方落到平台顶面的判定：

- 支持水平移动。
- 支持跳跃。
- 支持下落时落到平台顶面。
- 支持记录 `groundPlatformId`。

当前暂不包含：

- 墙体侧向碰撞。
- 斜坡。
- 单向平台穿透。
- 头顶碰撞。
- Cocos 物理组件接入。

这些内容后续步骤再单独确认并实现。
