# 敌人巡逻与接触伤害说明

## 1. 目标

第 15 步把基础敌人行为放入核心玩法层。敌人可以按配置进行水平巡逻，并在碰到玩家时触发接触伤害。预览和 Cocos 接入层只展示敌人位置与状态，不再自行判断接触伤害。

## 2. 当前行为

- 敌人初始化时记录 `patrolOriginX`。
- 敌人使用 `moveSpeed` 和 `patrolDistance` 左右巡逻。
- 敌人到达巡逻边界后会反向。
- 被击败的敌人不再巡逻，也不会造成接触伤害。
- 敌人与玩家判定区域相交时，会按 `contactDamage` 扣除玩家生命。
- 接触伤害复用玩家受伤后的无敌时间，避免同一段接触连续扣血。

## 3. 配置来源

敌人基础属性来自 `assets/resources/configs/enemy-slime.json`：

- `moveSpeed`：巡逻速度。
- `patrolDistance`：相对出生点左右移动的距离。
- `contactDamage`：接触伤害。
- `score`：击败得分。

敌人的受击判定尺寸来自关卡配置中的 `combat.enemyBounds`。

## 4. 边界

当前暂不包含：

- 追踪玩家。
- 跳跃或飞行敌人。
- 多敌人不同配置表。
- 敌人主动攻击或发射子弹。
- 路径点系统。

这些内容后续步骤再单独确认并实现。
