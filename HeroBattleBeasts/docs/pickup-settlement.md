# 自动拾取与结算冻结说明

## 1. 目标

第 16 步把道具拾取和胜负后的输入冻结放入核心运行时流程。预览和 Cocos 接入层只展示拾取结果，不再自行判断玩家是否碰到道具。

## 2. 自动拾取

当前行为：

- `updatePickups` 会检测玩家碰撞体和未收集道具碰撞体是否相交。
- 金币会增加 `coins`，并把道具标记为已收集。
- 武器强化会设置 `weaponBoostUntil`，持续时间来自武器配置。
- 已收集道具不会重复生效。

拾取判定尺寸来自关卡配置中的 `combat.pickupBounds`。

## 3. 结算冻结

`GameRuntime.step` 在状态不是 `playing` 时会直接返回当前视图模型，不再推进：

- 玩家移动和跳跃。
- 开火和子弹。
- 敌人巡逻和接触伤害。
- 自动拾取。

手动调试命令仍保留为运行时方法，但正常帧推进不会在胜利或失败后继续改变核心状态。

## 4. 边界

当前暂不包含：

- 结算菜单交互。
- 复活流程。
- 关卡重开 API。
- 存档奖励结算。

这些内容后续步骤再单独确认并实现。
