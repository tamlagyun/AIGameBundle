# 需求变更记录

本文件只追加记录，不覆盖历史条目。

## REQ-0.1 — 2026-07-13

- 状态：已确认
- 游戏中文名确定为《鲫鱼吃鲤鱼》，英文工程标识为 `FishEatFish`。
- 确定横屏 2D、单机 PvE、有限大海域和 AI 鱼群方向。
- 确定普通撕咬、冲刺撕咬、击败得经验和升级增加生命上限。
- 确定微信、抖音、Android、iOS、HarmonyOS 平台目标与统一适配器边界。
- 确定正式 UI 和美术不得使用 SVG 拼凑，AI 提示词须逐次确认并版本化存档。
- 确定项目卫生、性能、配置版本和存档版本基线。

## REQ-0.2 — 2026-07-13

- 状态：实施中
- 增加首张正式海洋地图显示和玩家小鲫鱼显示。
- 玩家使用 6 帧 Sprite Sheet 加工结果播放循环游泳动画，不接入视频生成模型。
- 增加 WASD、方向键和屏幕左侧拖动式虚拟摇杆输入。
- 增加海域边界限制和镜头跟随。
- 正式摇杆 UI 美术不在本需求内，避免未经提示词确认制作视觉资源。

## REQ-0.3 — 2026-07-13

- 状态：用户指定采用 v2 背景
- 地图视角由侧视海洋背景改为水中正上方垂直向下俯视海底的平面 2D 视角。
- 明确地图不得出现水面、天空、地平线、侧视剖面或斜俯视透视。
- 海底中央保持开阔，装饰和障碍主要放置在边缘区域。
- 多轮正交俯视提示词未得到合格结果；用户决定停止继续生成，明确指定 `ART-PROMPT-0001 v2` 作为当前正式地图背景。
