# 《鲫鱼吃鲤鱼》UI 布局基线

## 目标

本阶段采用横屏 1280×720 的安全区布局，交互结构参考移动 MOBA 的成熟布局逻辑，但不复制任何具体游戏的美术资源。

## 控件位置

- `JoystickControlRoot`：整体通过 Cocos `Widget` 固定到安全区左边 60 px、下边 35 px，尺寸 220×220；底盘、操纵帽和触摸事件全部属于该容器。
- 摇杆中心操纵帽：初始位于底盘中心，拖动范围限制在底盘半径内。
- `ActionControlsRoot`：整体尺寸 360×190，通过 `Widget` 固定到安全区右边 30 px、下边 30 px。
- 普通撕咬按钮和冲刺撕咬技能按钮：只使用 `ActionControlsRoot` 内部局部坐标，不再分别对齐屏幕边缘。

## 交互约定

- 摇杆拖动持续输出 `InputCommand.move`，松手后回中并停止移动。
- 玩家鱼儿使用移动向量计算 `atan2(y, x)` 角度，支持 360° 转向；松手后保持最后朝向。
- 普通攻击按钮触发 `InputCommand.basicAttack`，按钮按下和释放均提供视觉反馈。
- 技能按钮触发 `InputCommand.skill`，技能冷却期间显示不可用状态并阻止重复触发。
- 控件全部挂在 `SafeAreaRoot/InputLayer`，不随海底世界缩放；刘海屏、平板和折叠屏通过安全区适配。
- 正式美术资源使用 PNG，禁止用 SVG 或 Graphics 拼凑替代；每个正式资源均须登记提示词版本、加工步骤和 `.meta`。
