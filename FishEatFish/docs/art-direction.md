# 美术方向规范

状态：初始化约束，首次生成正式资源前须由用户确认具体风格提示词。

## 基础方向

- 明亮、友好、轻量的 2D 卡通海洋风格。
- 鱼类轮廓清晰，角色与背景保持足够明度和色相对比。
- 同批角色保持统一视角、描边粗细、光照方向和身体比例。
- UI 采用适合触摸操作的清晰大轮廓，信息层级优先于装饰。
- 角色、图标和特效按资源用途要求透明背景；场景背景按横屏构图生成。

## 技术规范

- 正式导入格式使用 PNG 等 Cocos 兼容位图，不使用 SVG。
- 角色动画状态至少预留 `idle`、`swim`、`bite`、`dashBite`、`hurt`、`death`。
- 同一角色的全部动画默认统一生成朝右侧视图，并在提示词与资源登记中明确写出；左右显示通过水平翻转复用。
- 鱼配置使用 `artFacingDirection` 声明运行时统一美术方向，并用 `animationArtFacingDirections` 登记每组导入原图的实际方向；加载时通过 SpriteFrame UV 翻转归一，不改动逻辑移动与服务器攻击方向。
- 存在明显非对称设计时必须单独生成并登记，不得复用普通水平翻转。
- 原始生成图、裁切图和最终游戏资源分目录保存，正式资源必须登记来源和加工步骤。
- Sprite Sheet 不得仅凭列数和行数判定可安全切分；正式导入前必须检查每帧 Alpha 包围盒不接触画布边缘，也不得包含相邻画格的角色残片。发现跨格但各主体仍独立时，使用扩展画格和最大连通主体重新提取，并保持同批帧统一缩放、身体基准和透明边距。
